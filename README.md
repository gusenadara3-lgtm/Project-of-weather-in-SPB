## Описание проекта
В этом проекте мы реализовали систему прогнозирования и анализа погоды по CSV‑файлу.
В файле WEATHER_SPB.csv хранится температура за каждый день годов 2025-2026: дата и числовое значение температуры. Программа читает этот файл, считает разные статистики и показывает работу нескольких структур данных: префиксные суммы, бинарное дерево поиска, очередь и стек.

## Структура проекта

## Основной функционал программы

Программа последовательно выполняет следующие шаги:

1. **Загрузка данных из CSV:**
  Класс `WeatherAnalyzer` читает файл `WEATHER_SPB.csv` и сохраняет:
  список дат,
  список температур,
  список номеров месяцев, а затем автоматически строит массив префиксных сумм по температурам.
  
2. **Средняя температура по месяцам (префиксные суммы):**
  Метод `get_month_average`:
  находит границы каждого месяца в списке;
  использует префиксные суммы для быстрого вычисления суммарной температуры за месяц;
  делит на количество дней и выводит среднюю температуру по каждому месяцу.

3. **Самый холодный и самый тёплый день (линейный поиск):**
  Метод `get_min_max_day`:
  проходит по всему списку температур;
  линейным поиском находит минимум и максимум;
  возвращает соответствующие даты и значения.
  
4. **Сортировка месяцев по средней температуре:**
  Метод `sort_months`:
  получает словарь `месяц → средняя температура`;
  сортирует месяцы по средней температуре с помощью **сортировки выбором**;
  выводит месяцы от самых холодных к самым тёплым.

5. **Бинарное дерево поиска по температуре:**
  Для работы с «жаркими» днями реализованы классы:
  `Node` — узел дерева: хранит температуру, список дат, левый и правый потомок;
  `WeatherBST` — бинарное дерево поиска по температуре.
  
  В `WeatherAnalyzer` метод `get_hot_days_bst` по всем температурам строит дерево, а затем вызывает рекурсивный обход `get_hot_days`, который возвращает все дни, где температура строго выше заданного порога. 
  
  **Дополнительно** реализован метод `get_tree_levels`, который выводит температуры по уровням дерева (обход в ширину) для демонстрации его структуры.

6. **Сглаживание температурного ряда (скользящее среднее, очередь):**
  Метод `smooth_temps`:
  сохраняет текущий список температур в стек `undo_stack`;
  создаёт очередь `deque` с максимальной длиной 7;
  проходит по температурному ряду, добавляя на каждом шаге значение в очередь, и считает среднее по окну;
  заменяет исходный ряд на сглаженный и пересчитывает префиксные суммы.
  
7. **Стек для отмены последнего преобразования:**
  Метод `undo_last_change`:
  проверяет наличие сохранённого состояния в стеке;
  если оно есть — извлекает предыдущий список температур, восстанавливает его и заново строит префиксные суммы;
  если стек пуст — выводит сообщение, что отменять нечего.
 

## Используемые структуры данных и алгоритмы

В проекте задействованы следующие структуры и алгоритмы:

**Списки:** для хранения дат, температур и номеров месяцев.

**Префиксные суммы:** массив `prefix` для быстрого подсчета сумм температур на любом интервале за время O(1).

**Линейный поиск:** для нахождения минимальной и максимальной температуры за O(n).

**Сортировка выбором:** для ранжирования месяцев по средней температуре за O(m^2).

**Бинарное дерево поиска:** для хранения дней по температуре и рекурсивного поиска дней выше заданного порога.

**Очередь:** для реализации скользящего среднего с плавающим окном в 7 дней.

**Стек:** для сохранения предыдущих версий температурного ряда и отмены последнего сглаживания.

**Классы и ООП:** структуры `Node`, `WeatherBST` и `WeatherAnalyzer` для логичного разделения кода по задачам.
